# 建筑视觉交付加速工具 - 产品 SPEC（v1.0 MVP 版）

## 1. 业务背景与核心问题

### 1.1 行业痛点

在建筑与空间设计的交付环节，存在三类高频但低效的需求，传统流程无法支撑：

| 业务场景 | 现有痛点 | 用户核心诉求 |
|----------|----------|--------------|
| 白模转竣工效果图 | BIM / 无材质模型，传统渲染流程周期长（平均一周以上）、改一次沟通成本极高 | 快速、低成本地将白模转化为高质量竣工效果图 |
| 效果图多版本迭代 | 通用 AI 出图结构易变形，无法批量生成同一建筑的多风格 / 多场景版本 | 锁定建筑主体结构，快速生成多个合规、稳定的效果图版本 |
| 施工场布分阶段可视化 | 仅靠静态图纸无法直观展示不同施工阶段的现场布置，汇报交底效率低 | 一键生成不同施工阶段的可视化效果，支撑方案沟通与汇报 |

**关键洞察**：最耗时的不是「渲染技术本身」，而是「等客户反馈 + 反复修改」的沟通循环。痛点核心是「精力消耗」而非「金钱成本」。

**结构变形零容忍**：任何细节不一致都视为失败，对 AI 出图质量要求极高。

### 1.2 市场现状

- **现有工具**：用户主要使用专业建筑行业工具，以渲染图生成为主
- **市场空白**：市面上少有同等施工图 / 白模类产品，竞争对手少
- **技术壁垒**：结构锁定能力为行业首创，具有数据飞轮效应（用户越多 → 数据越多 → 模型越好）

---

## 2. 产品定位

**产品定位**：BIM / 方案图 → 视觉交付的加速层工具

**核心价值**：在用户现有模型 / 图纸基础上，不替代建模 / 渲染软件，快速生成结构稳定、可用于正式交付的效果图与可视化内容

**核心差异**：专为建筑交付场景优化，解决传统流程和通用 AI 无法兼顾的「效率、成本、结构一致性」问题

**核心定位说明**：本工具是「交付加速层」，而非替代用户现有的设计 / 渲染流程。

## 3. 目标用户

| 业务场景 | 操作角色 | 使用特点 |
|----------|----------|----------|
| 白模转效果图 | 效果图制作专员 | 批量处理，专业操作 |
| 多版本迭代 | 设计负责人 / 甲方提修改意见后跟进的人 | 根据反馈快速调整 |
| 施工场布可视化 | 项目经理 / 技术人员 | 向非技术背景的甲方汇报使用 |

## 4. 核心功能流程（MVP 版）

### 4.1 交互设计原则

**单页引导式布局**：整个操作流程在一个页面内完成，左侧固定图片预览区，右侧动态切换操作面板。用户始终可见当前处理的图片，减少页面跳转带来的认知负担。

**精简流程**：从传统多步骤页面跳转简化为 6 步引导，降低用户学习成本。

### 4.4 三个场景详细说明

#### 场景 A：白模 → 竣工效果图

| 环节 | 说明 |
|------|------|
| 输入 | BIM 白模图片（当前仅支持图片格式，其他格式后期扩展） |
| 结构识别重点 | 建筑主体轮廓、层数、门窗位置、屋顶形态 |
| 图像生成目标 | 自动补材质、环境、光影，生成写实竣工效果 |
| 局部再生成 | 可修改材质、颜色、门窗样式等单构件 |
| 典型用户 | 效果图制作专员 |

#### 场景 B：效果图 → 多版本迭代

| 环节 | 说明 |
|------|------|
| 输入 | 已有效果图 |
| 结构识别重点 | 建筑结构（来自效果图，可能本身有瑕疵，需锁定） |
| 图像生成目标 | 换风格 / 换场景 / 换氛围，保持结构不变 |
| 局部再生成 | 可修改风格单构件（如换屋顶样式、换墙面颜色） |
| 典型用户 | 设计负责人（根据甲方反馈快速调整） |

#### 场景 C：施工场布 → 分阶段可视化

| 环节 | 说明 |
|------|------|
| 输入 | 2D 施工场布图纸 |
| 结构识别重点 | 施工区域划分、构件布局、机械设备位置 |
| 图像生成目标 | 按施工阶段生成不同进度的现场效果 |
| 局部再生成 | 可修改单构件（如更换塔吊位置、调整围挡范围） |
| 典型用户 | 项目经理 / 技术人员（向甲方汇报） |

## 5. AI 能力与生成提示词

### 5.1 结构识别与锁定

| 属性 | 说明 |
|------|------|
| 触发时机 | 用户上传图片后自动触发 |
| 用户操作 | 无需手动标注，AI 自动识别并保持结构 |
| 结构保持 | 生成时强制要求建筑轮廓、形体、比例、层数与原图一致 |
| 干预机制 | 局部再生成时可输入修改指令，仅修改指定区域 |

### 5.2 图像生成提示词

#### 批量生成提示词模板

```
照片级真实感建筑渲染图，photorealistic rendering, ultra detailed, DSLR相机拍摄，专业建筑摄影，[选项]。

建筑轮廓、建筑形体、建筑比例、建筑层数必须与参考图完全一致。

HDR高动态范围光照，8K超高清细节，建筑外观精美，材质真实，光影自然。
```

#### 局部再生成提示词模板

```
照片级真实感建筑渲染图，photorealistic rendering, ultra detailed, DSLR相机拍摄，专业建筑摄影。

【修改指令】：[用户输入的指令]

【关键要求】：
1. 严格保持原图建筑轮廓、建筑形体、建筑比例、建筑层数完全一致
2. 只修改用户指定的区域或构件，其他部分必须保持100%不变
3. 材质、光影、细节风格与原图保持整体一致性
4. HDR高动态范围光照，8K超高清细节
5. 禁止添加、删除或移动任何建筑元素
```

### 5.3 图像生成规格

| 属性 | 说明 |
|------|------|
| 生成数量 | 固定 4 张候选图 |
| 清晰度选项 | HD (1280×720) / FHD (1920×1080) / 2K (2560×1440) / 4K (3840×2160) |
| 生成质量 | 照片级真实感，HDR 光照，8K 超高清细节 |

### 5.4 视频生成（架构预留）

| 属性 | 说明 |
|------|------|
| 当前状态 | **暂不做，架构预留** |
| 预计场景 | 打包输出阶段，支持施工场布分阶段可视化视频 |
| 触发方式 | 待定（用户主动选择或自动生成） |

## 6. 技术约束与应对方案

### 6.1 MVP 版本技术约束

| 技术难点 | 对业务的影响 | 应对方案 |
|----------|--------------|----------|
| 结构一致性（核心难点） | 效果图 / 施工场布图变形，无法用于正式交付 | 生成提示词强制要求建筑轮廓、形体、比例、层数与原图一致。用户确认前可干预，确认后锁定。 |
| 结果随机性 | 单张出图质量不稳定，用户无法依赖 | 多结果批量生成机制，提供 4 个候选结果，支持对比筛选 |
| 高质量输出成本高 | 单用户成本过高，无法规模化 | 分级清晰度策略：HD/FHD/2K/4K，按需选择 |
| 输入质量强依赖 | 白模 / 图纸质量差，生成效果不稳定 | 输入预处理模块，系统自动检测并提示（允许但警告），同时明确输入规范 |

### 6.2 输入规范

| 输入类型 | 当前支持格式 | 质量要求 |
|----------|--------------|----------|
| BIM 白模图片 | 图片（JPG/PNG） | 图片清晰，建筑结构可辨识 |
| 已有效果图 | 图片（JPG/PNG） | 图片清晰，风格明确 |
| 施工场布图纸 | 2D 图片（JPG/PNG） | 图纸清晰，标注可读 |

**说明**：当前仅支持图片格式，其他 BIM 格式（.rvt / .pln / IFC）后期可能扩展。

## 7. 功能边界（MVP 版）

### 7.1 支持的功能

- ✅ 场景选择（三个场景均可选）
- ✅ 图片上传（拖拽/点击上传，支持 JPG/PNG）
- ✅ AI 结构识别（自动执行，无需手动标注）
- ✅ 生成选项选择（风格/场景/角度/天气单选，清晰度可选）
- ✅ 批量生成（固定 4 张候选图，照片级真实感）
- ✅ 候选图选择（点击选中）
- ✅ 局部再生成（输入指令修改，AI 保持其他部分不变）
- ✅ 快捷指令（蓝色玻璃幕墙、增加绿植、黄昏光照、更换材质）
- ✅ 导出高清图（支持不同清晰度下载）
- ✅ 单页引导式交互（左侧预览 + 右侧操作面板）

### 7.2 暂不做的功能（MVP 后迭代）

- ❌ 跨图拼接局部内容（将 A 图的天空嫁接到 B 图）
- ❌ 视频生成（占位，架构预留）
- ❌ 多张图纸批量处理（当前仅支持单张上传）
- ❌ 结构锁定模板跨用户共享
- ❌ 精细室内设计（软装、材质精细化调整）
- ❌ 复杂图像编辑（替代 Photoshop）
- ❌ 专业渲染器功能（光线追踪级材质精控）
- ❌ BIM 协同 / 项目管理系统

## 9. 成功标准（MVP 版）

### 9.1 MVP 验证目标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 种子用户数 | ≥10 人 | 能持续使用产品的真实用户 |
| 导出成功率 | ≥80%（8/10） | 用户生成后成功导出高清图的比例 |
